# 📑 KeepOrCut 커밋 컨벤션

## 1. 메시지 구조
> **type(scope): subject**

## 2. Type
- **feat**: 새로운 기능 추가
- **fix**: 버그 수정
- **docs**: 문서 수정
- **style**: 코드 포맷팅 (로직 변경 없음)
- **refactor**: 코드 리팩토링
- **test**: 테스트 코드 추가/수정
- **chore**: 빌드 설정, .gitignore 수정 등

## 3. Scope
- **back**: Spring Boot 관련
- **front**: Next.js 관련
- **root**: 프로젝트 공통

## 4. 규칙
- 제목은 한글로 작성
- 끝에 마침표(.) 찍지 않기
